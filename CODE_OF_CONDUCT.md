# Code of Conduct

## Purpose

This project exists to make Korean NOTAM tooling easier to build, study, and improve.

We want the repository to be useful to aviation developers, researchers, students, and contributors with different levels of domain knowledge. That only works if discussion stays respectful, practical, and focused on the work.

## Expected Behavior

Contributors and maintainers are expected to:

- assume good intent and communicate clearly
- focus criticism on code, behavior, or evidence
- share context when raising bugs, risks, or safety concerns
- respect different experience levels and language backgrounds
- keep operational and aviation claims grounded in sources or reproducible examples

## Unacceptable Behavior

The following is not acceptable in project spaces:

- harassment, intimidation, or personal attacks
- discriminatory or insulting language
- bad-faith arguing, trolling, or repeated derailment
- posting secrets, private data, or sensitive operational material
- presenting unverified data as authoritative for safety-critical use

## Scope

This Code of Conduct applies to issues, pull requests, discussions, email related to the project, and other project-managed communication channels.

## Maintainer Responsibilities

Maintainers may remove, edit, or reject comments, issues, commits, or other contributions that conflict with this Code of Conduct or create unnecessary legal, operational, or safety risk for the project.

## Reporting

If you experience or witness a serious problem, contact:

- allofdaniel@naver.com

Include links, screenshots, and any relevant context so the report can be reviewed quickly.

## Enforcement

Maintainers may warn, pause, or block participation depending on severity and pattern of behavior. Good-faith technical disagreement is welcome. Repeated harmful behavior is not.
